# 1주차 — IPv4 주소 체계와 서브네팅

- **일시**: 2026-04-06 (월)
- **시간**: 1~3시간
- **방식**: 실습 중심 (명령어 직접 입력하면서 개념 설명)
- **목표**: 자기 IP를 보면서 IPv4 구조 이해 → 서브네팅 계산까지

---

## 흐름 요약

```
내 IP 확인 → IP 구조 설명 → 클래스/사설IP → 통신 확인 → 서브네팅 계산
```

---

## 1. 시작 — 내 IP부터 확인

노트북에서 바로 실행

**Windows**
```
ipconfig
ipconfig /all
```

**Mac / Linux**
```
ip addr
ifconfig
```

확인할 것:
- `IPv4 Address` — 내 IP 주소
- `Subnet Mask` — 서브넷 마스크
- `Default Gateway` — 게이트웨이 (라우터 IP)

> **강의 포인트**: 화면에 뜬 숫자들이 오늘 배울 내용 전부다. 하나씩 뜯어보자.

---

## 2. IP 주소 구조

나온 IP를 칠판에 적고 같이 분석

- 32비트 = 4옥텟 = 점(`.`)으로 구분한 숫자 4개
- 각 옥텟: 0 ~ 255

```
예: 192.168.0.15
    └─┘ └─┘ └┘ └┘
    1옥 2옥 3옥 4옥
```

---

## 3. 클래스와 사설 IP

`ipconfig` 결과 보면서 직접 해당 클래스 찾기

| 클래스 | 첫 번째 옥텟 | 기본 서브넷 마스크 |
|--------|------------|------------------|
| A      | 1 ~ 126    | 255.0.0.0 (/8)   |
| B      | 128 ~ 191  | 255.255.0.0 (/16)|
| C      | 192 ~ 223  | 255.255.255.0 (/24)|
| D      | 224 ~ 239  | (멀티캐스트)      |
| E      | 240 ~ 255  | (예약)            |

**사설 IP 범위 (암기)**
```
A:  10.0.0.0       ~ 10.255.255.255
B:  172.16.0.0     ~ 172.31.255.255
C:  192.168.0.0    ~ 192.168.255.255
```

> 대부분 `192.168.x.x` → 클래스 C 사설 IP. 왜 사설인지 설명.

---

## 4. 통신 확인 — ping

```
ping 127.0.0.1          # 루프백 (자기 자신)
ping [게이트웨이 IP]     # 공유기까지 통신 확인
ping 8.8.8.8            # 구글 DNS (인터넷 통신 확인)
```

결과 보면서:
- 응답 오면 통신 O
- `TTL` 값 간략 설명 (라우터 홉 줄어드는 것)
- `Request timed out` → 차단 또는 응답 없음

---

## 5. 서브넷 마스크와 CIDR

`ipconfig` 에서 본 `Subnet Mask: 255.255.255.0` 을 직접 해석

- 서브넷 마스크 = 어디까지가 네트워크인지 구분
- CIDR 표기: `255.255.255.0` = `/24`

```
IP:   192.168.0.15
MSK:  255.255.255.0  (/24)
      └────────────┘ └───┘
       네트워크 부분   호스트
```

---

## 6. 서브네팅 계산

### 공식
| 항목 | 계산 |
|------|------|
| 블록 크기 | 2^(호스트 비트 수) |
| 네트워크 주소 | IP AND 서브넷 마스크 |
| 브로드캐스트 | 네트워크 주소 + 블록 크기 - 1 |
| 사용 가능 호스트 | 블록 크기 - 2 |

### 예제 (같이 풀기)
```
192.168.1.100 / 26

/26 → 호스트 비트 6개 → 블록 크기 64

64 단위 블록:  0~63 / 64~127 / 128~191 ...
100은 64~127 블록

네트워크 주소:  192.168.1.64
브로드캐스트:   192.168.1.127
호스트 범위:    192.168.1.65 ~ 192.168.1.126  (62개)
```

### 자기 IP로 직접 해보기
`ipconfig` 에서 나온 본인 IP + 서브넷 마스크로 동일하게 계산

## 핵심 정리

- **ipconfig로 확인**: IP, 서브넷 마스크, 게이트웨이
- **클래스 구분**: 첫 번째 옥텟 기준
- **사설 IP 3가지**: 10.x / 172.16~31.x / 192.168.x
- **서브네팅**: CIDR → 블록 크기 → 해당 블록 → 네트워크/브로드캐스트
