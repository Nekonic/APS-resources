## 공개키
암호화키 =! 복호화키

1976년에 나옴

각 개인에게 필요한 키는 자신의 개인키와 공개키 2개 뿐

| | 장점 | 단점 |
| - | - | - |
| 대칭키 암호 | 익숙, 빠른 속도 | 키공유, 키관리 |
| 공개키 암호 | 키공유,키관리 해결 | 느린 속도, 공개키의 안전성 |
- 공인인증서:<br>어떤 사람의 공개키가 그 사람 것이 맞다고 증명(신뢰할 수 있는 제 3자가 서명)해주는 문서
- 하이브리드 암호:<br>메세지는 대칭키로 암호화, 대칭키의 암호의 키는 공개키 암호로 암호화
