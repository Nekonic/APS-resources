## 1과목. 네트워크 보안

### 1.1 OSI 7계층 & TCP/IP

| OSI 계층 | 이름 | TCP/IP | 주요 프로토콜 |
|---------|------|--------|-------------|
| 7 | 응용(Application) | 응용 | HTTP, FTP, SMTP, DNS, SNMP |
| 6 | 표현(Presentation) | 응용 | SSL/TLS |
| 5 | 세션(Session) | 응용 | NetBIOS |
| 4 | 전송(Transport) | 전송 | TCP, UDP |
| 3 | 네트워크(Network) | 인터넷 | IP, ICMP, ARP, RARP |
| 2 | 데이터링크(Data Link) | 네트워크 접근 | Ethernet, PPP |
| 1 | 물리(Physical) | 네트워크 접근 | - |

### 1.2 TCP vs UDP

| 항목 | TCP | UDP |
|------|-----|-----|
| 연결 | 연결 지향 (3-way handshake) | 비연결 |
| 신뢰성 | 보장 (재전송) | 미보장 |
| 순서 | 보장 | 미보장 |
| 속도 | 느림 | 빠름 |
| 용도 | HTTP, FTP, SMTP | DNS, DHCP, 스트리밍 |

### 1.3 TCP 3-way Handshake

```
Client ──SYN──────────────► Server
Client ◄──SYN+ACK────────── Server
Client ──ACK──────────────► Server
```

### 1.6 주요 네트워크 공격

#### SYN Flooding
- TCP 3-way handshake에서 SYN만 대량 전송
- Half-open connection 소진 → 서비스 거부
- 대응: SYN Cookie, 방화벽 임계값 설정

#### IP Spoofing
- 출발지 IP 주소 위조
- 대응: Ingress/Egress 필터링

#### Smurf Attack
- 위조된 피해자 IP로 브로드캐스트 ICMP Echo 전송
- 네트워크 전체가 피해자에게 ICMP Reply 전송
- 대응: 브로드캐스트 라우팅 차단, ICMP 제한

#### Fragmentation Attack
- IP 단편화(Fragment)를 이용한 침입 탐지 우회
- Teardrop: 잘못된 offset으로 커널 크래시 유발

#### ARP Spoofing
- ARP Reply 위조로 MAC 테이블 오염
- 중간자 공격(MITM) 가능
- 대응: Static ARP, ARP Inspection

## 2과목. 시스템 보안

### 2.1 리눅스 파일 권한

```
-rwxr-xr--  1  user  group  size  date  filename
 │││ │││ │││
 │││ │││ └── Others: r--  (4)
 │││ └────── Group:  r-x  (5)
 └────────── Owner:  rwx  (7)
```

**권한 값**
- r (read) = 4
- w (write) = 2
- x (execute) = 1

**특수 권한**
- SetUID (4xxx): 실행 시 파일 소유자 권한으로 실행 → `chmod 4755`
- SetGID (2xxx): 실행 시 파일 그룹 권한으로 실행 → `chmod 2755`
- Sticky Bit (1xxx): 디렉토리에서 타인 파일 삭제 방지 → `chmod 1777`

**위험한 SetUID 파일 탐색**
```bash
find / -perm -4000 -type f 2>/dev/null
```

## 3과목. 애플리케이션 보안

### 3.1 웹 취약점 (OWASP Top 10 기준)

#### SQL Injection
- DB 쿼리 조작으로 인증 우회, 데이터 탈취
- 예시: `' OR '1'='1' --`
- 대응: **준비된 구문(Prepared Statement)**, 입력값 검증, WAF

#### XSS (Cross-Site Scripting)
- 악성 스크립트를 웹 페이지에 삽입
- **Stored XSS**: DB에 저장 → 모든 방문자 피해
- **Reflected XSS**: URL 파라미터 → 즉시 반환
- **DOM-based XSS**: 클라이언트 DOM 조작
- 대응: 출력 인코딩, CSP, HTTPOnly 쿠키

#### CSRF (Cross-Site Request Forgery)
- 사용자 권한을 도용해 의도치 않은 요청 전송
- 대응: **CSRF Token**, SameSite 쿠키, Referer 검증

#### 파일 업로드 취약점
- 웹쉘 업로드 후 원격 코드 실행
- 대응: 파일 확장자 검증, MIME 타입 검증, 저장 경로 분리

#### 디렉토리 트래버설 (Path Traversal)
- `../../../etc/passwd` 등으로 접근 불가 파일 열람
- 대응: 입력값 정규화, 허용 경로 화이트리스트

#### 커맨드 인젝션 (Command Injection)
- OS 명령어 삽입 실행
- 예시: `; cat /etc/passwd`
- 대응: 쉘 명령어 미사용, 입력값 이스케이프

### 3.2 인증/세션 보안

**세션 고정 공격 (Session Fixation)**
- 공격자가 미리 설정한 세션 ID를 사용자에게 강제
- 대응: 로그인 후 세션 ID 재발급

**세션 하이재킹 (Session Hijacking)**
- 유효한 세션 토큰 탈취
- 대응: HTTPS, HTTPOnly, Secure 쿠키 플래그

**쿠키 속성**
| 속성 | 설명 |
|------|------|
| Secure | HTTPS에서만 전송 |
| HttpOnly | JS에서 접근 불가 (XSS 방어) |
| SameSite | 크로스사이트 요청 제한 (CSRF 방어) |
| Domain | 쿠키 유효 도메인 |
| Path | 쿠키 유효 경로 |
| Expires/Max-Age | 만료 시간 |

### 3.3 암호화

#### 대칭키 암호화
| 알고리즘 | 블록 크기 | 키 길이 | 비고 |
|---------|---------|--------|------|
| DES | 64bit | 56bit | 취약 (사용 금지) |
| 3DES | 64bit | 112/168bit | DES 3회 적용 |
| AES | 128bit | 128/192/256bit | 현재 표준 |
| SEED | 128bit | 128bit | 국내 표준 |
| ARIA | 128bit | 128/192/256bit | 국내 표준 |

#### 비대칭키 암호화
| 알고리즘 | 기반 | 용도 |
|---------|------|------|
| RSA | 소인수분해 | 암호화, 전자서명 |
| DSA | 이산대수 | 전자서명 |
| ECC | 타원곡선 | 모바일 환경 |
| DH | 이산대수 | 키 교환 |

#### 해시 알고리즘
| 알고리즘 | 출력 길이 | 비고 |
|---------|---------|------|
| MD5 | 128bit | 취약 (충돌 가능) |
| SHA-1 | 160bit | 취약 (사용 금지 권고) |
| SHA-256 | 256bit | 현재 권장 |
| SHA-512 | 512bit | 높은 보안 |
| HAS-160 | 160bit | 국내 표준 |

#### PKI (공개키 기반 구조)
- **CA (인증기관)**: 인증서 발급/폐기
- **RA (등록기관)**: 신원 확인 후 CA에 인증 요청
- **CRL (인증서 폐기 목록)**: 폐기된 인증서 목록
- **OCSP**: 실시간 인증서 상태 확인

### 3.4 SSL/TLS Handshake

```
Client                          Server
  │── ClientHello ─────────────► │  (지원 암호화 목록, 난수)
  │◄─ ServerHello ──────────────  │  (선택된 암호화, 난수)
  │◄─ Certificate ──────────────  │  (서버 인증서)
  │◄─ ServerHelloDone ──────────  │
  │── ClientKeyExchange ────────► │  (pre-master secret 전송)
  │── ChangeCipherSpec ─────────► │
  │── Finished ─────────────────► │
  │◄─ ChangeCipherSpec ──────────  │
  │◄─ Finished ──────────────────  │
  │══════ 암호화 통신 시작 ══════════  │
```

## 4과목. 보안 관리

### 4.1 방화벽 (Firewall)

**유형**
- **패킷 필터링**: IP/포트 기반 필터링, 빠르지만 단순
- **스테이트풀 인스펙션**: 연결 상태 추적, 더 정교
- **애플리케이션 게이트웨이(프록시)**: L7 레벨 검사, 느리지만 안전
- **차세대 방화벽(NGFW)**: DPI + IPS + 앱 인식

**DMZ (비무장지대)**
```
인터넷 ──► [외부 방화벽] ──► DMZ (웹서버, 메일서버) ──► [내부 방화벽] ──► 내부망
```

### 4.2 IDS / IPS

| 항목 | IDS | IPS |
|------|-----|-----|
| 역할 | 탐지 및 경고 | 탐지 + 차단 |
| 위치 | 네트워크 탭 | 인라인 |
| 오탐 영향 | 알림만 | 정상 트래픽 차단 가능 |

**탐지 방식**
- **시그니처 기반**: 알려진 공격 패턴 탐지 (오탐 적음, 미탐 가능)
- **이상 행위 기반**: 정상 기준선 대비 이상 탐지 (오탐 많음, 미탐 적음)

### 4.3 VPN

| 유형 | 프로토콜 | 계층 |
|------|---------|------|
| IPSec VPN | IPSec | 3계층 (IP) |
| SSL VPN | SSL/TLS | 7계층 (응용) |
| PPTP | PPTP | 2계층 |
| L2TP | L2TP/IPSec | 2계층 |

**IPSec 구성요소**
- **AH (Authentication Header)**: 무결성 + 인증 (암호화 없음)
- **ESP (Encapsulating Security Payload)**: 무결성 + 인증 + **암호화**
- **IKE**: 키 교환 프로토콜
- **SA (Security Association)**: 보안 연결 설정 정보

### 4.4 접근 통제 모델

| 모델 | 설명 | 특징 |
|------|------|------|
| DAC (임의적 접근통제) | 소유자가 권한 부여 | 유연하지만 취약 |
| MAC (강제적 접근통제) | 보안 레이블 기반 (관리자) | 엄격, 군/정부 |
| RBAC (역할기반 접근통제) | 역할에 따라 권한 부여 | 기업 환경 적합 |
| ABAC (속성기반 접근통제) | 다양한 속성 조합 | 세밀한 제어 |

**Bell-LaPadula 모델 (기밀성)**
- No Read Up: 낮은 등급은 높은 등급 문서 읽기 불가
- No Write Down: 높은 등급은 낮은 등급 문서 쓰기 불가

**Biba 모델 (무결성)**
- No Read Down: 높은 등급은 낮은 등급 문서 읽기 불가
- No Write Up: 낮은 등급은 높은 등급 문서 쓰기 불가

### 4.5 보안 관련 법률/인증

**국내 법률**
- 정보통신망 이용촉진 및 정보보호 등에 관한 법률 (정보통신망법)
- 개인정보 보호법
- 전자서명법
- 정보보호산업의 진흥에 관한 법률

**보안 인증/기준**
| 인증 | 설명 |
|------|------|
| ISMS-P | 정보보호 및 개인정보보호 관리체계 (국내) |
| ISO 27001 | 정보보호 관리체계 (국제) |
| CC (공통평가기준) | IT 제품 보안성 평가 |
| PIMS | 개인정보보호 관리체계 (ISMS-P에 통합) |

### 4.6 개인정보보호

**개인정보 처리 원칙 (개인정보 보호법)**
1. 목적 명확화

## 5과목. 침해사고 대응

### 5.1 침해사고 대응 절차 (PICERL)

1. **준비(Preparation)**: 대응 팀 구성, 도구 준비, 정책 수립
2. **식별(Identification)**: 침해사고 탐지 및 확인
3. **봉쇄(Containment)**: 피해 확산 방지 (격리)
4. **박멸(Eradication)**: 악성코드 제거, 취약점 패치
5. **복구(Recovery)**: 시스템 정상 복구, 모니터링
6. **교훈(Lessons Learned)**: 분석 보고서, 재발 방지

### 5.2 디지털 포렌식

**기본 원칙**
- **무결성**: 증거 변조 방지 (해시값 검증)
- **연계 보관성(Chain of Custody)**: 증거 수집~분석 전 과정 기록
- **재현성**: 동일한 환경에서 동일한 결과
- **신속성**: 휘발성 데이터 우선 수집

**휘발성 데이터 수집 순서 (휘발성 높은 순)**
1. CPU 레지스터, 캐시
2. 메모리 (RAM)
3. 네트워크 연결 상태
4. 실행 중인 프로세스
5. 열린 파일
6. 디스크 (비휘발성)

**해시를 이용한 무결성 검증**
```bash

```

## 빠른 암기 포인트

### 주요 공격 vs 대응

| 공격 | 핵심 대응 |
|------|---------|
| SQL Injection | Prepared Statement |
| XSS | 출력 인코딩, CSP |
| CSRF | CSRF Token, SameSite |
| SYN Flooding | SYN Cookie |
| ARP Spoofing | Static ARP, Dynamic ARP Inspection |
| 무차별 대입(Brute Force) | 계정 잠금 정책, 다중 인증(MFA) |
| 스니핑 | 암호화(HTTPS, SSH) |
| 스푸핑 | 인증서, MAC/IP 필터링 |

### 암호화 용도 구분

| 구분 | 알고리즘 | 키 수 |
|------|---------|-------|
| 대칭키 | AES, DES, SEED | 1개 (공유키) |
| 비대칭키 | RSA, ECC | 2개 (공개키 + 개인키) |
| 해시 | SHA-256, MD5 | 없음 (단방향) |
