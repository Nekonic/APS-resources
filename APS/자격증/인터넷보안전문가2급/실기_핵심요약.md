## 1. 계정 보안 ★★★★★

### root 권한 탈취 계정 찾기
```bash
cat /etc/passwd              # 확인 파일
awk -F: '($3==0)' /etc/passwd   # UID=0 계정 탐지
cat /etc/passwd | grep ":0:"    # 동일
```

### 계정 잠금/해제
```bash
passwd -l 계정명    # 잠금
passwd -u 계정명    # 해제
```

### 자동 로그아웃
```bash
vi /etc/profile
TMOUT=30       # 30초
export TMOUT
```

### 패스워드 정책
```bash
vi /etc/login.defs
PASS_MAX_DAYS  30    # 최대 사용 기간
PASS_MIN_LEN   8     # 최소 길이
```

> `/etc/profile` = TMOUT / `/etc/login.defs` = 패스워드 정책

### shadow 파일 필드
```
계정:암호:lastchg:min:max:warn:inactive:expire
              ③     ④   ⑤    ⑥     ⑦
```
| ③ | 마지막 변경일 | `chage -d` |
| ④ | 최소 사용 기간 | `chage -m` |
| ⑤ | 최대 사용 기간 | `chage -M` |
| ⑥ | 만료 경고 기간 | `chage -W` |
| ⑦ | 만료 후 유예 기간 | `chage -I` |

```bash
chage -m 7 -M 90 -W 14 -I 30 계정명
chage -l 계정명    # 정책 확인 (잠금 아님!)
```

**패스워드 필드 특수값**
```
* → 로그인 불가   !! → 계정 잠금   + → NIS
```

## 2. 파일 권한 ★★★★★

### 숫자 암기
```
rwx=7  rw-=6  r-x=5  r--=4  -wx=3  -w-=2  --x=1  ---=0
```

### 특수권한
```
4xxx = SetUID → rws(owner)  실행 시 소유자 권한
2xxx = SetGID → rws(group)  실행 시 그룹 권한
1xxx = Sticky → rwt         본인 파일만 삭제 가능

s = 실행권한 있음   S = 실행권한 없음(비정상, rw-=6)
```

### 명령어
```bash
chmod 644 /etc/service                         # 기출 단골
chmod 622 /etc/service                         # 기출 단골
chown user:group 파일                           # 소유자+그룹 동시
chown -R user:group /dir                       # 하위 포함
chmod -R 755 /dir                              # 하위 포함
find /data -type f -exec chmod 644 {} \;       # 파일만
chmod u-s 파일                                  # SetUID 제거
chmod o+t /dir                                 # Sticky Bit (o+s 아님!)
chmod g-w,o-rwx 파일                            # 콤마로 연결
find / -perm /6000 -type f 2>/dev/null         # SetUID/GID 탐색
find / -user nobody 2>/dev/null                # 특정 사용자 파일
```

## 4. 서비스 보안 설정 ★★★★★

### SSH
```bash
vi /etc/ssh/sshd_config        # 파일명 주의 (sshd_config, _ 사용)
Port 2222                      # 포트 변경 (= 없음)
PermitRootLogin no
PasswordAuthentication no
MaxAuthTries 3

systemctl restart sshd         # 적용 (sshd, ssh 아님)
```

### FTP
```bash
vi /etc/vsftpd/vsftpd.conf     # vsftpd (vsftp 아님)
anonymous_enable=NO            # 익명 접근 차단

vi /etc/ftpaccess
loginfails 3                   # 로그인 실패 횟수 제한
```

### TCP Wrapper
```bash
vi /etc/hosts.allow            # 허용 (먼저 확인)
sshd : ALL

vi /etc/hosts.deny             # 차단 (나중 확인)
ALL : ALL
```

### Apache
```bash
vi /etc/httpd/conf/httpd.conf
ServerTokens Prod              # 버전 숨김 (Full → Prod)
ServerSignature Off
Options -Indexes               # 디렉토리 목록 차단
Timeout 300
```

## 5. iptables ★★★★★

### 기본 템플릿
```bash
iptables -P INPUT DROP
iptables -P FORWARD DROP
iptables -P OUTPUT ACCEPT

iptables -A INPUT -s 차단IP -j DROP                               # 최우선

iptables -A INPUT -i lo -j ACCEPT
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

iptables -A INPUT -p tcp --dport 22 -s 192.168.0.0/24 -j ACCEPT
iptables -A INPUT -p tcp -m multiport --dports 80,443 -j ACCEPT
iptables -A INPUT -p udp --dport 53 -j ACCEPT
iptables -A INPUT -p icmp -j DROP

iptables -A INPUT -p tcp --dport 23 -j LOG --log-prefix "TELNET: "
iptables -A INPUT -p tcp --dport 23 -j DROP
```

### SSH 브루트포스 방어
```bash
iptables -A INPUT -p tcp --dport 22 -m state --state NEW -m recent --set
iptables -A INPUT -p tcp --dport 22 -m state --state NEW -m recent --update --seconds 60 --hitcount 4 -j DROP
```

### 관리
```bash
iptables -L -n -v --line-numbers    # 번호 포함 조회
iptables -D INPUT 3                 # 번호로 삭제
iptables -F && iptables -X && iptables -Z   # 완전 초기화
iptables-save > /etc/iptables/rules.v4
```

**오타 주의**: `ESTABLISHED` / `ACCEPT` / `--dports`(복수) / `-m recent` / `-p tcp` 필수

## 6. ICMP / sysctl ★★★★☆

```bash
# ICMP Echo 무시 (기출 정답 — iptables 아님)
sysctl -w net.ipv4.icmp_echo_ignore_all=1
echo 1 > /proc/sys/net/ipv4/icmp_echo_ignore_all

# SYN Cookie (SYN Flooding 방어)
sysctl -w net.ipv4.tcp_syncookies=1

# 영구 적용
vi /etc/sysctl.conf
net.ipv4.icmp_echo_ignore_all=1
sysctl -p
```

## 8. 로그 분석 ★★★★☆

```bash
# 브루트포스 IP 탐지
grep "Failed password" /var/log/auth.log | awk '{print $11}' | sort | uniq -c | sort -rn | head -10

# 웹 접속 IP 상위
awk '{print $1}' /var/log/apache2/access.log | sort | uniq -c | sort -rn | head -5

# 특정 상태코드
grep " 404 " /var/log/apache2/access.log

tail -f /var/log/auth.log    # 실시간 모니터링
```

**주요 로그 파일**
```
/var/log/auth.log   인증 (SSH, sudo)
/var/log/secure     인증 (RHEL/CentOS)
/var/log/messages   시스템, su 기록
/var/log/wtmp       로그인 이력 (last)
/var/log/btmp       실패 기록 (lastb)
/var/log/lastlog    마지막 로그인 (lastlog)
```

## 9. tcpdump ★★★☆☆

```bash

tcpdump -r /tmp/out.pcap -nv
```

## 10. 서술형 핵심 개념 ★★★★☆

| 개념 | 설명 |
|------|------|
| **Shell Code** | 버퍼오버플로우 등으로 삽입, 쉘 실행시켜 공격자 명령 수행 |
| **허니팟** | 의도적 취약 시스템으로 공격자 유인·분석 |
| **IDS** | 탐지만, Out-of-band 배치 |
| **IPS** | 탐지+차단, In-line 배치 |
| **오탐(FP)** | 정상을 공격으로 탐지 |
| **미탐(FN)** | 공격을 정상으로 탐지 |
| **SYN Cookie** | SYN Flooding 대응, 쿠키로 정상 클라이언트 검증 |
| **Null Session** | 윈도우 익명 IPC$ 접근, RestrictAnonymous=1로 차단 |
| **침해사고 대응** | 준비→식별→봉쇄→박멸→복구→교훈 |
| **포렌식 휘발성** | 레지스터→RAM→네트워크→프로세스→파일→디스크 |
