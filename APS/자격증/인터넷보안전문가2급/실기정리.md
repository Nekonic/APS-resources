## 1. 리눅스 시스템 보안

### 1.1 계정 및 패스워드 관리

```bash
# 사용자 생성/삭제
useradd -m -s /bin/bash username
userdel -r username

# 패스워드 설정
passwd username

# 계정 잠금 / 잠금 해제
usermod -L username   # 잠금
usermod -U username   # 해제
passwd -l username    # 잠금
passwd -u username    # 해제

# 패스워드 만료 설정
chage -M 90 username       # 최대 사용 기간 90일
chage -m 7 username        # 최소 사용 기간 7일
chage -W 14 username       # 만료 14일 전 경고
chage -l username          # 패스워드 정책 확인

# 패스워드 정책 설정 (PAM)
# /etc/pam.d/common-password
password requisite pam_pwquality.so minlen=8 dcredit=-1 ucredit=-1 ocredit=-1 lcredit=-1
```

**패스워드 복잡도 옵션**
| 옵션 | 설명 |
|------|------|
| `minlen=8` | 최소 길이 |
| `dcredit=-1` | 숫자 최소 1개 |
| `ucredit=-1` | 대문자 최소 1개 |
| `lcredit=-1` | 소문자 최소 1개 |
| `ocredit=-1` | 특수문자 최소 1개 |

### 1.2 파일/디렉토리 권한

```bash
# 권한 변경
chmod 755 file          # rwxr-xr-x
chmod u+x file          # 소유자 실행권한 추가
chmod go-w file         # 그룹/기타 쓰기 권한 제거
chmod -R 644 /dir       # 디렉토리 재귀 적용

# 소유자 변경
chown user:group file
chown -R user:group /dir

# 특수 권한
chmod 4755 /usr/bin/file    # SetUID
chmod 2755 /usr/bin/file    # SetGID
chmod 1777 /tmp             # Sticky Bit

# 위험한 SetUID 파일 탐색 및 제거
find / -perm -4000 -type f 2>/dev/null
chmod -s /path/to/file      # SetUID 제거

# 쓰기 가능한 파일 탐색
find / -perm -o+w -type f 2>/dev/null

# umask 설정 (기본 생성 권한 마스크)
umask 022   # 파일: 644, 디렉토리: 755
umask 027   # 파일: 640, 디렉토리: 750
```

## 2. iptables 방화벽

### 2.1 기본 구조

```
테이블: filter / nat / mangle
체인:   INPUT / OUTPUT / FORWARD (filter)
       PREROUTING / POSTROUTING (nat)
타겟:   ACCEPT / DROP / REJECT / LOG
```

### 2.2 기본 명령어

```bash
# 규칙 조회
iptables -L -n -v              # filter 테이블
iptables -L -n -v --line-numbers  # 줄 번호 포함
iptables -t nat -L -n -v       # nat 테이블

# 규칙 초기화
iptables -F                    # 모든 규칙 삭제
iptables -X                    # 사용자 정의 체인 삭제
iptables -Z                    # 카운터 초기화

# 기본 정책 설정
iptables -P INPUT DROP         # 기본 차단
iptables -P FORWARD DROP
iptables -P OUTPUT ACCEPT
```

### 2.3 규칙 추가/삭제

```bash
# 추가 (Append / Insert)
iptables -A INPUT -p tcp --dport 22 -j ACCEPT
iptables -I INPUT 1 -p tcp --dport 80 -j ACCEPT  # 1번 위치에 삽입

# 삭제
iptables -D INPUT -p tcp --dport 22 -j ACCEPT    # 규칙 직접 지정
iptables -D INPUT 3                               # 번호로 삭제
```

### 2.4 자주 사용하는 규칙 패턴

```bash
# 루프백 허용
iptables -A INPUT -i lo -j ACCEPT

# 기존 연결 허용 (Stateful)
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# 특정 IP 차단
iptables -A INPUT -s 192.168.1.100 -j DROP

# 특정 IP 대역 허용
iptables -A INPUT -s 192.168.0.0/24 -j ACCEPT

# 포트 허용
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT
iptables -A INPUT -p tcp --dport 22 -s 10.0.0.0/8 -j ACCEPT

# 포트 범위 허용
iptables -A INPUT -p tcp --dport 8080:8090 -j ACCEPT

# 다중 포트
iptables -A INPUT -p tcp -m multiport --dports 80,443,8080 -j ACCEPT

# ICMP (ping) 제한
iptables -A INPUT -p icmp --icmp-type echo-request -j DROP

# SYN Flooding 방어
iptables -A INPUT -p tcp --syn -m limit --limit 1/s --limit-burst 4 -j ACCEPT

# 포트 스캔 방어
iptables -A INPUT -p tcp --tcp-flags ALL NONE -j DROP   # NULL 스캔
iptables -A INPUT -p tcp --tcp-flags ALL ALL -j DROP    # XMAS 스캔

# 로깅
iptables -A INPUT -p tcp --dport 23 -j LOG --log-prefix "TELNET BLOCKED: "
iptables -A INPUT -p tcp --dport 23 -j DROP

# NAT (IP 마스커레이딩)
iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE

# 포트 포워딩
iptables -t nat -A PREROUTING -p tcp --dport 80 -j DNAT --to-destination 192.168.1.10:8080
```

### 2.5 규칙 저장/복원

```bash
# 저장
iptables-save > /etc/iptables/rules.v4

# 복원
iptables-restore < /etc/iptables/rules.v4
```

## 3. 네트워크 분석

### 3.1 tcpdump

```bash
# 기본 캡처
tcpdump -i eth0                           # 인터페이스 지정
tcpdump -i any                            # 모든 인터페이스
tcpdump -n                                # IP 주소로 표시 (DNS 조회 안 함)
tcpdump -nn                               # IP + 포트 번호로 표시
tcpdump -v / -vv / -vvv                   # 상세 출력

# 필터링
tcpdump host 192.168.1.1                  # 특정 호스트
tcpdump src 192.168.1.1                   # 출발지
tcpdump dst 192.168.1.1                   # 목적지
tcpdump port 80                           # 포트
tcpdump src port 80                       # 출발지 포트
tcpdump tcp                               # 프로토콜
tcpdump icmp

# 조합 필터
tcpdump host 192.168.1.1 and port 80
tcpdump not port 22
tcpdump 'tcp[tcpflags] & tcp-syn != 0'   # SYN 패킷

# 파일 저장/읽기
tcpdump -w capture.pcap -i eth0
tcpdump -r capture.pcap

# 패킷 수 제한
tcpdump -c 100 -i eth0

# ASCII/HEX 출력
tcpdump -A -i eth0     # ASCII
tcpdump -X -i eth0     # HEX + ASCII
```

### 3.2 netstat / ss

```bash
# 연결 상태 확인
netstat -an             # 모든 연결 (IP:포트)
netstat -antp           # TCP + PID/프로세스명
netstat -anup           # UDP + PID/프로세스명
netstat -l              # LISTEN 상태만
netstat -s              # 통계 정보
netstat -r              # 라우팅 테이블

# ss (netstat 대체)
ss -antp
ss -ltnp                # LISTEN 중인 TCP

# 특정 포트 사용 프로세스 확인
lsof -i :80
lsof -i tcp:22
```

### 3.3 nmap

```bash
# 기본 스캔
nmap 192.168.1.1
nmap 192.168.1.0/24

# 스캔 유형
nmap -sS 192.168.1.1    # SYN 스캔 (Half-open, 스텔스)
nmap -sT 192.168.1.1    # TCP Connect 스캔
nmap -sU 192.168.1.1    # UDP 스캔
nmap -sN 192.168.1.1    # NULL 스캔
nmap -sX 192.168.1.1    # XMAS 스캔
nmap -sF 192.168.1.1    # FIN 스캔
nmap -sP 192.168.1.0/24 # Ping 스캔 (호스트 탐지)

# 포트 지정
nmap -p 80,443 192.168.1.1
nmap -p 1-1024 192.168.1.1
nmap -p- 192.168.1.1    # 전체 포트

# 버전/OS 탐지
nmap -sV 192.168.1.1    # 서비스 버전
nmap -O 192.168.1.1     # OS 탐지
nmap -A 192.168.1.1     # 모두 포함

# 출력 저장
nmap -oN result.txt 192.168.1.1   # 일반 텍스트
nmap -oX result.xml 192.168.1.1   # XML
```

## 4. 웹 취약점 분석 및 대응

### 4.1 SQL Injection 탐지 및 대응

**취약한 코드 (PHP)**
```php
// 취약
$query = "SELECT * FROM users WHERE id='" . $_GET['id'] . "'";

// 안전 (Prepared Statement)
$stmt = $pdo->prepare("SELECT * FROM users WHERE id = ?");
$stmt->execute([$_GET['id']]);
```

**공격 패턴 탐지 (로그에서)**
```
' OR '1'='1
' OR 1=1--
' UNION SELECT
; DROP TABLE
' AND SLEEP(5)
```

**WAF 규칙 (nginx)**
```nginx
# /etc/nginx/nginx.conf
if ($query_string ~* "union.*select.*\(") { return 403; }
if ($query_string ~* "concat.*\(") { return 403; }
```

### 4.2 XSS 대응

**입력값 검증 및 출력 인코딩**
```php
// 입력 필터
$input = strip_tags($_POST['input']);

// 출력 인코딩 (HTML 엔터티)
echo htmlspecialchars($input, ENT_QUOTES, 'UTF-8');
```

**CSP 헤더 설정 (Apache)**
```apache
Header set Content-Security-Policy "default-src 'self'; script-src 'self'"
```

**쿠키 보안 설정**
```php
setcookie('session', $id, [
    'secure'   => true,   // HTTPS만
    'httponly' => true,   // JS 접근 차단
    'samesite' => 'Strict'
]);
```

### 4.3 Apache 웹서버 보안 설정

```apache
# /etc/apache2/conf-enabled/security.conf

# 버전 정보 숨기기
ServerTokens Prod
ServerSignature Off

# 디렉토리 목록 비활성화
Options -Indexes

# 심볼릭 링크 비활성화
Options -FollowSymLinks

# .htaccess 오버라이드 차단
AllowOverride None

# HTTP 메서드 제한
<LimitExcept GET POST HEAD>
    deny from all
</LimitExcept>

# 헤더 보안
Header always set X-Frame-Options "SAMEORIGIN"
Header always set X-XSS-Protection "1; mode=block"
Header always set X-Content-Type-Options "nosniff"
Header always set Strict-Transport-Security "max-age=31536000"
```

### 4.4 Nginx 보안 설정

```nginx
# /etc/nginx/nginx.conf

# 버전 정보 숨기기
server_tokens off;

# 요청 크기 제한
client_max_body_size 10m;
client_body_timeout 12;
client_header_timeout 12;

# 보안 헤더
add_header X-Frame-Options "SAMEORIGIN";
add_header X-XSS-Protection "1; mode=block";
add_header X-Content-Type-Options "nosniff";
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains";

# 허용 메서드 제한
if ($request_method !~ ^(GET|HEAD|POST)$) {
    return 405;
}
```

## 5. 암호화 실습

### 5.1 OpenSSL

```bash
# 대칭키 암호화 (AES-256)
openssl enc -aes-256-cbc -in plaintext.txt -out encrypted.bin -k password
openssl enc -aes-256-cbc -d -in encrypted.bin -out decrypted.txt -k password

# RSA 키 생성
openssl genrsa -out private.key 2048          # 개인키 생성
openssl rsa -in private.key -pubout -out public.key  # 공개키 추출

# 인증서 서명 요청(CSR) 생성
openssl req -new -key private.key -out cert.csr

# 자체 서명 인증서 생성
openssl req -new -x509 -key private.key -out cert.crt -days 365

# 인증서 정보 확인
openssl x509 -in cert.crt -text -noout

# 해시 계산
openssl dgst -md5 file.txt
openssl dgst -sha256 file.txt

# Base64 인코딩/디코딩
openssl base64 -in file.bin -out file.b64
openssl base64 -d -in file.b64 -out file.bin
```

### 5.2 GPG

```bash
# 키 생성
gpg --gen-key
gpg --full-generate-key

# 키 목록 확인
gpg --list-keys
gpg --list-secret-keys

# 파일 암호화 (수신자 공개키 사용)
gpg -e -r recipient@email.com file.txt

# 파일 복호화 (본인 개인키 사용)
gpg -d file.txt.gpg

# 전자서명
gpg --sign file.txt            # 서명 포함 파일 생성
gpg --clearsign file.txt       # 명문 서명
gpg --detach-sign file.txt     # 분리 서명 (.sig)

# 서명 검증
gpg --verify file.txt.sig file.txt
```

## 8. 주요 서비스 보안 설정

### 8.1 FTP 보안 (vsftpd)

```bash
# /etc/vsftpd.conf

anonymous_enable=NO          # 익명 로그인 금지
local_enable=YES
write_enable=YES
local_umask=022
chroot_local_user=YES        # 홈 디렉토리 감금
ftpd_banner=Welcome          # 버전 정보 숨기기
xferlog_enable=YES           # 전송 로그 활성화
ssl_enable=YES               # FTPS 사용
```

### iptables 자주 나오는 패턴

```bash
# 1. 기본 보안 정책 (DROP all, allow necessary)
iptables -P INPUT DROP
iptables -P FORWARD DROP
iptables -P OUTPUT ACCEPT
iptables -A INPUT -i lo -j ACCEPT
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
iptables -A INPUT -p tcp --dport 22 -j ACCEPT
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# 2. 특정 IP 차단 (블랙리스트)
iptables -I INPUT 1 -s 1.2.3.4 -j DROP

# 3. Ping 허용/차단
iptables -A INPUT -p icmp --icmp-type echo-request -j ACCEPT  # 허용
iptables -A INPUT -p icmp -j DROP                              # 전체 차단

# 4. 포트 포워딩
iptables -t nat -A PREROUTING -p tcp --dport 80 -j REDIRECT --to-port 8080
```
