Cursor AI 코드 에디터 취약점, MCP 파일 악의적 교체로 원격 코드 실행 가능

날짜/작성자

2025-08-07

25_김영빈

핵심 내용

CVE-2025-54136으로 추적되는 Cursor AI 코드 에디터의 심각한 취약점이 공개되었다. 이 취약점은 Model Context Protocol(MCP) 설정 파일(.cursor/rules/mcp.json)을 악의적으로 교체함으로써, 공격자가 사용자의 승인 이후에도 변조된 MCP 파일을 통해 지속적으로 원격 코드 실행(RCE)을 할 수 있다는 점이 특징이다.

공격 시나리오는 다음과 같다. 먼저 무해한 MCP 설정을 GitHub 공유 레포지토리에 추가한 후, 사용자가 이를 한 번 승인하게 만들고, 이후 해당 MCP 설정을 악성 페이로드로 교체한다. Cursor는 최초 승인 이후에는 파일의 내용이 바뀌어도 재승인 없이 계속 신뢰하기 때문에, 공격자가 백도어 스크립트 등 악성 코드를 사용자의 컴퓨터에서 원격·지속적으로 실행할 수 있다.

이 결함은 AI 코드 에디터 및 LLM 기반 도구에서 신뢰 모델의 핵심적인 약점을 드러내며, 협업 리포지토리와 자동화 개발 환경이 실제로 공급망 공격(supply chain attack) 및 지적 재산권(코드·데이터) 탈취에 노출될 수 있음을 시사한다. MCPoison이라고 명명된 해당 취약점은 2025년 7월 16일 공개되었으며, Cursor는 1.3 버전에서 MCP 변경 시 매번 승인을 요구하도록 패치하였다.

이와 동시에, AI 개발 환경에 특화된 공급망 공격, 프롬프트 인젝션(prompt injection), 악성 파일 템플릿, 모델 포이즈닝(model poisoning), LLM 보안 실패 등 다양한 공격기법도 연구·발견되고 있으며, 보안 커뮤니티에서는 새로운 패러다임의 보안 관리와 AI 보안 모범사례가 필요하다는 목소리가 커지고 있다.

키워드 정리

CVE-2025-54136: Cursor AI의 MCP 구성 신뢰 결함, 원격 코드 실행 취약점

MCPoison: MCP 설정 교체 기반의 코드 실행 공급망 공격

공급망 공격(Supply Chain Attack): 개발 도구·환경 취약점을 노려 조직 전체에 악성코드를 전파하는 공격

프롬프트 인젝션(Prompt Injection): 명령어/조건문 삽입을 통한 LLM 내부 규칙 우회 또는 악성 행동 유발 방식
