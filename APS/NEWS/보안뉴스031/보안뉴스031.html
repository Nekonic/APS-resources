Microsoft SharePoint Server 원격 코드 실행 취약점 (CVE-2025-53770) 분석

날짜/작성자

2025-07-31

25_김영빈

핵심 내용

2025년 7월, 'ToolShell'이라는 별칭으로 알려진 심각한 제로데이 취약점(CVE-2025-53770)이 Microsoft SharePoint Server에서 발견되었다.

이 취약점은 인증되지 않은 원격 공격자가 서버의 완전한 제어권을 획득할 수 있게 하는 원격 코드 실행(RCE) 결함이다. CVSS 3.1 기준 9.8점(치명적)으로 평가되어 매우 높은 위험도를 가진다.

공격의 핵심 원인은 신뢰할 수 없는 데이터에 대한 '부적절한 역직렬화' 과정에 있다. 공격자는 조작된 데이터를 서버로 전송하여 서버가 이를 처리하는 과정에서 악성 코드를 실행시킨다.

특히 'ToolShell' 공격은 단순히 코드를 실행하는 것을 넘어, 서버의 암호화 키를 탈취하는 기능을 포함한다. 이는 공격자가 초기 패치가 적용된 이후에도 지속적으로 서버에 접근할 수 있는 '백도어'를 확보하게 만들어 피해를 심화시킨다. 이미 Storm-2603과 같은 공격 그룹이 이 취약점을 랜섬웨어 배포에 악용하고 있는 정황이 포착되었다.

미국 CISA는 이 취약점을 '알려진 악용된 취약점(KEV)' 목록에 추가하며 즉각적인 대응을 촉구했다. Microsoft는 긴급 보안 업데이트를 배포했으며, 추가적인 방어 조치로 AMSI(Antimalware Scan Interface) 통합 기능 활성화를 권고하고 있다.

키워드 정리

CVE-2025-53770 ('ToolShell'): Microsoft SharePoint Server의 치명적인 원격 코드 실행 제로데이 취약점이다.

원격 코드 실행(RCE): 공격자가 원격지에서 대상 시스템에 임의의 명령이나 코드를 실행하는 공격 유형이다.

제로데이 취약점: 보안 패치가 발표되기 전에 외부에 알려져 실제 공격에 악용되고 있는 보안 결함이다.

역직렬화(Deserialization): 데이터 스트림이나 파일 형태의 데이터를 메모리 상의 객체로 변환하는 과정. 이 과정의 보안 결함은 심각한 취약점으로 이어진다.

CISA KEV: 미국 사이버보안 및 인프라 보안국(CISA)이 관리하는, 실제 공격에 활발히 악용되는 것으로 확인된 취약점 목록이다.
