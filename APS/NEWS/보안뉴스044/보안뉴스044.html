CL0P 랜섬웨어 그룹, Oracle EBS 제로데이 취약점 악용해 수십 개 조직 침해

날짜/작성자

2025-10-27

25_김영빈

핵심 내용

공격 개요

2025년 8월 9일부터 Cl0p 랜섬웨어 그룹이 Oracle E-Business Suite(EBS)의 제로데이 취약점 CVE-2025-61882를 악용하여 수십 개 조직을 침해했다. Google Threat Intelligence Group과 Mandiant의 공동 보고서에 따르면, 일부 과거 Cl0p 데이터 갈취 캠페인은 수백 명의 피해자를 냈으며, 이러한 대규모 제로데이 캠페인이 사이버범죄의 일상적인 특징이 되고 있다.

취약점 상세

CVE-2025-61882는 CVSS 점수 9.8의 치명적 등급 취약점으로, Oracle Concurrent Processing 컴포넌트에 영향을 미친다. 이 취약점은 인증 없이 HTTP를 통해 원격에서 악용 가능하며, 영향받는 버전은 12.2.3부터 12.2.14까지이다. 공격자는 이 취약점을 통해 대상 시스템에 완전한 제어 권한을 획득할 수 있다.

공격 체인 분석

공격자는 다음과 같은 정교한 다단계 공격 체인을 사용했다:

1. SSRF 악용: /OA_HTML/configurator/UiServlet 엔드포인트에서 서버 측 요청 위조를 통해 초기 침투 지점 확보

2. CRLF 주입: URL 페이로드에 캐리지 리턴/라인 피드 시퀀스를 주입하여 요청 프레이밍 조작 및 공격자 제어 헤더 삽입

3. 인증 우회: HTTP 연결 재사용(keep-alive)을 활용하여 동일 TCP 소켓으로 요청 파이프라이닝

4. XSL 템플릿 주입: /OA_HTML/SyncServlet 컴포넌트의 취약점을 악용하여 원격 코드 실행 달성 및 Template Preview 기능을 통한 XSL 페이로드 트리거

사용된 멀웨어

XSL 페이로드에 내장된 두 가지 Java 페이로드 체인이 발견되었다:

• GOLDVEIN.JAVA: 2024년 12월 Cleo 소프트웨어 제품 공격에서 처음 발견된 다운로더 GOLDVEIN의 Java 변종으로, C2 서버에서 2단계 페이로드 수신 가능

• SAGEGIFT → SAGELEAF → SAGEWAVE: Oracle WebLogic 서버용으로 맞춤 설계된 Base64 인코딩 로더 SAGEGIFT가 인메모리 드로퍼 SAGELEAF를 실행하고, 이는 다시 악성 Java 서블릿 필터 SAGEWAVE를 설치하여 알려지지 않은 차기 단계 멀웨어가 포함된 암호화된 ZIP 아카이브 설치를 허용한다. 주요 페이로드는 FIN11 백도어 GOLDTOMB의 cli 모듈과 일부 중첩된다.

협박 캠페인

9월 29일부터 공격자는 수백 개의 침해당한 제3자 계정에서 기업 임원들에게 대량의 이메일 캠페인을 시작했다. 이 계정들의 자격증명은 인포스틸러 멀웨어 로그 구매를 통해 언더그라운드 포럼에서 구입된 것으로 추정된다. 이메일은 Oracle EBS 애플리케이션을 침해하고 민감한 데이터를 탈취했다고 주장하며, 데이터 유출을 막는 대가로 특정되지 않은 금액의 몸값을 요구했다. 현재까지 피해 조직들은 Cl0p 데이터 유출 사이트에 게시되지 않았으며, 이는 공격자가 수주간 기다린 후 게시하는 과거 Cl0p 공격 패턴과 일치한다.

위협 행위자 분석

Google은 이 공격을 추적된 위협 그룹에 공식적으로 귀속시키지 않았지만, Cl0p 브랜드 사용이 주목할 만하다고 지적했다. 공격 후 툴링은 FIN11 캠페인에서 사용된 멀웨어(GOLDVEIN 및 GOLDTOMB)와 중첩을 보이며, 최근 협박 이메일에 사용된 침해 계정 중 하나는 이전에 FIN11이 사용한 적이 있다. 널리 사용되는 엔터프라이즈 애플리케이션의 제로데이 취약점을 악용한 후 수주 후 대규모 브랜드 협박 캠페인을 펼치는 패턴은 FIN11에 역사적으로 귀속된 활동의 특징이며, 다른 위협 행위자들에게도 전략적 이점을 제공할 수 있다.

대응 조치

Oracle은 10월 4일 긴급 패치를 발표했으며, 10월 11일에는 CVE-2025-61884를 해결하는 추가 패치를 배포했다. 미국 CISA는 10월 6일 이 취약점을 Known Exploited Vulnerabilities(KEV) 목록에 추가했다. 영국 NCSC와 싱가포르 CSA 등 여러 정부 기관이 긴급 경보를 발령했다. 조직들은 최소한 2023년 10월 Critical Patch Update를 사전 설치한 후 즉시 2025년 10월 패치를 적용해야 한다. 민감한 데이터를 저장하는 공개 노출 애플리케이션과 어플라이언스를 표적으로 삼는 것은 공격자가 측면 이동에 시간과 리소스를 투입할 필요가 없어 데이터 절도 작전의 효율성을 높인다.

키워드 정리

CVE-2025-61882: Oracle E-Business Suite의 치명적(CVSS 9.8) 제로데이 취약점으로, 인증 없이 원격 코드 실행이 가능하다

Cl0p 랜섬웨어: 2020년부터 활동한 랜섬웨어 그룹으로, Accellion FTA, GoAnywhere MFT, MOVEit MFT, Cleo LexiCom 등의 제로데이를 대규모로 악용한 이력이 있다

SSRF (Server-Side Request Forgery): 서버가 공격자가 조작한 요청을 내부 또는 외부 리소스로 전송하도록 강제하는 취약점

CRLF 주입: HTTP 헤더에 캐리지 리턴(\r)과 라인 피드(\n) 문자를 주입하여 요청 스머글링이나 응답 분할을 일으키는 공격 기법

XSL 템플릿 주입: XSLT 처리 과정에서 악의적인 XSL 템플릿을 주입하여 임의 코드를 실행하는 공격

FIN11: 피싱 이메일 캠페인과 Cl0p 랜섬웨어 배포와 연관된 금융 동기 위협 그룹

데이터 갈취 (Data Extortion): 데이터를 암호화하는 대신 또는 암호화와 함께 민감한 데이터를 탈취한 후 공개하지 않는 대가로 몸값을 요구하는 공격 방식

인포스틸러 (Infostealer): 피해자 시스템에서 로그인 자격증명, 쿠키, 저장된 비밀번호 등을 수집하는 멀웨어

제로데이 (Zero-day): 벤더가 인지하지 못하고 패치가 존재하지 않는 취약점으로, 공개된 시점부터 패치가 배포되기까지 공격자가 악용할 수 있다

GOLDVEIN/SAGEGIFT/SAGEWAVE: 이번 공격에서 사용된 맞춤형 멀웨어 프레임워크로, 다단계 페이로드 전달 및 지속성 확보에 사용된다
